# Cross-Document Consistency Dimensions

What to compare when checking multiple documents in the same matter.

---

## 1. Factual Assertions

**What**: Facts stated as true in one document should not contradict facts in another.

**Examples of conflicts**:
- Different effective dates for the same statute
- Different contract values or percentages
- Different party counts or names
- Different timeline of events

**Severity**: Major by default. Critical if the contradiction affects a legal conclusion.

---

## 2. Terminology

**What**: The same concept should use the same term across all documents.

**Examples of drift**:
- "개인정보" in one doc vs "개인 정보" in another
- "Intellectual Property" in one doc vs "IP Rights" in another
- "갑" vs "매도인" for the same party

**Severity**: Minor (cosmetic). Major if the drift creates ambiguity.

---

## 3. Party Designations

**What**: The same party should have consistent names and roles.

**Examples of conflicts**:
- Party named differently across documents
- Role designation inconsistency (갑/을 reversed)
- Full name vs abbreviated name without clear mapping

**Severity**: Major.

---

## 4. Date References

**What**: Timeline must be internally consistent across documents.

**Examples of conflicts**:
- Contract date differs between the review report and the opinion
- Deadline stated differently in different documents
- Sequence of events contradicts between documents

**Severity**: Major.

---

## 5. Legal Conclusions

**What**: Conclusions reached in one document should not contradict conclusions in another.

**Examples of conflicts**:
- Research report concludes activity is permitted; opinion concludes it requires approval
- Different risk assessments for the same issue
- Contradictory recommendations

**Severity**: Critical. This is the most serious cross-document inconsistency.

---

## 6. Amounts & Numbers

**What**: Identical figures should match exactly.

**Examples of conflicts**:
- Different penalty amounts cited for the same statute
- Different threshold values
- Different fee calculations

**Severity**: Major. Critical if the amount is dispositive.

---

## Comparison Methodology

1. Extract key entities from each document (dates, parties, amounts, conclusions)
2. Create a comparison matrix: entity × document
3. For each entity, verify consistency across all documents
4. Flag inconsistencies with locations in both documents
5. Recommend which document to correct (or verify externally)
